# Glossary

This glossary centralizes key terms used across the APTITUDE RFCs.

## Roles

The entities (human or legal) participating in the ecosystem that perform specific functions.

<glossary::roles|theme=table>

## Components

The functional units, software modules, or subsystems operated by a Role to perform specific technical operations within the EUDI Wallet ecosystem.

<glossary::components|theme=table>

## Artifacts

Supporting technical objects, tokens, or proofs used to establish trust or state, distinct from user identity data.

<glossary::artifacts|theme=table>

## Credentials

The digital statements containing information about a User.

<glossary::credentials|theme=table>

## Protocols

The rules, sequences and profiles governing communication between Components and Roles.

<glossary::protocols|theme=table>

## Processes

The defined sequences of actions and interactions between Roles within specific Protocols.

<glossary::processes|theme=table>

## Formats

The syntax and data structures used to represent artifacts and credentials.

<glossary::formats|theme=table>

## Data Elements

Individual pieces of data, fields, or cryptographic values that are used within Credentials, Artifacts, or Protocols.

<glossary::data-elements|theme=table>
